Checker-Historie loggen

Dieses Kapitel behandelt CheckerHistoryLogger. CheckerHistoryLogger loggen die Ergebnisse beliebiger anderer Checker.

CheckerHistoryLogger

Ein CheckerHistoryLogger erzeugt Logfiles für die Ergebnisse beliebiger Checker. Der CheckerHistoryLogger wird selbst als Checker im Job definiert und ruft seinerseits den tatsächlichen Checker, dessen Name in den Parametern übergeben wurde, als Sub-Checker auf.

Danach erstellt der CheckerHistoryLogger eine Logdatei mit historische Werten des Sub-Checkers oder ergänzt eine schon vorhandene Logdatei mit den aktuell ermittelten Werten.

  Wichtig

Die Dll des Sub-Checkers muss sich (inkl. dort referenzierter Assemblies) im Plugin-Verzeichnis des aktuellen Jobs befinden.

  Hinweis

Die Logdateien der CheckerHistoryLogger finden Sie im SnapshotDirectory des aktuellen Jobs.

der Demo Job

So sieht der Demo-Job CheckWeatherHistory für den CheckerHistoryLogger im Betrieb aus:

Und so sieht die zugehörige JobDescription aus:

XML
Json
Copy
<?xml version="1.0" encoding="utf-8"?>
<JobDescription>
  <LogicalName>Check Weather History</LogicalName>
  <LogicalExpression>IS WeatherHistory</LogicalExpression>
  <Checkers type="array">
    <Checker>
      <LogicalName>WeatherHistory</LogicalName>
      <UserControlPath>Plugin\WeatherCheckerControl.dll</UserControlPath>
      <PhysicalPath>CheckerHistoryLogger.dll</PhysicalPath>
      <Parameters>
        <![CDATA[quiet|WeatherChecker.dll|erstellt Wettervorhersagen]]>
      </Parameters>
      <Trigger>
        <PhysicalPath>TimerTrigger.dll</PhysicalPath>
        <Parameters>M:60</Parameters>
      </Trigger>
    </Checker>
  </Checkers>
</JobDescription>

{
  "JobDescription": {
    "LogicalName": "Check Weather History",
    "LogicalExpression": "IS WeatherHistory",
    "Checkers": {
      "Checker": [
        {
          "LogicalName": "WeatherHistory",
          "UserControlPath": "Plugin\\WeatherCheckerControl.dll",
          "PhysicalPath": "CheckerHistoryLogger.dll",
          "Parameters": "quiet|WeatherChecker.dll|erstellt Wettervorhersagen",
          "Trigger": {
            "PhysicalPath": "TimerTrigger.dll",
            "Parameters": "M:60"
          }
        }
      ]
    }
  }
}

Über die vom CheckerHistoryLogger aufgerufene WeatherChecker.dll werden für den aktuellen Tag und die darauf folgendenden Tage Wettervorhersagen im Abstand von jeweils drei Stunden geholt und über das eigene UserControl WeatherCheckerControl.dll übersichtlich dargestellt.

  Hinweis

Der verwendete CheckerHistoryLogger arbeitet nur mit ToString(), da ihm der Typ des konkreten ReturnObjects des Sub-Checkers unbekannt ist.

Für ein spezifischeres Logging des ReturnObjects des SubCheckers können die Klasse CheckerHistoryLogger abgeleitet und die virtuellen Methoden SetupSubCheckerResultsLogging, LogSubCheckerData und ReadHistory überschrieben werden.
