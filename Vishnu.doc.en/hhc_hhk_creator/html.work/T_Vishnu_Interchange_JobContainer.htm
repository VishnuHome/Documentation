BreakWithResult Ein Job kann u.U. schon ein eindeutiges logisches Ergebnis haben, bevor alle Kinder ihre Verarbeitung beendet haben. Bei BreakWithResult=True werden diese dann abgebochen. Greift nicht, wenn ein JobTrigger gesetzt wurde.
BreakWithResultString String-Property zur Vermeidung von Serialisierungsfehlern wegen Groß- Kleinschreibung
Checkers Liste von externen Prüfroutinen für einen Job.
ConstantNodeUserControlPath Pfad zum dynamisch zu ladenden UserControl für eine Constant-SingleNode.
Format Das Format, in dem dieser Job gespeichert wurde (Json oder Xml).
InitNodes Bei True werden alle Knoten im Tree resettet, wenn dieser Knoten gestartet wird. Kann für Loops in Controlled-Jobs verwendet werden. Default: false.
InitNodesString String-Property zur Vermeidung von Serialisierungsfehlern wegen Groß- Kleinschreibung
IsVolatile Bei True wird zur Ergebnisermittlung im Tree Logical benutzt, bei False LastNotNullLogical; Default: false.
IsVolatileString String-Property zur Vermeidung von Serialisierungsfehlern wegen Groß- Kleinschreibung
JobConnectorUserControlPath Pfad zum dynamisch zu ladenden UserControl für einen JobConnector.
JobListUserControlPath Pfad zum dynamisch zu ladenden UserControl für eine JobList.
JobLogger Ein optionaler Logger, der dem Job zugeordnet ist oder null.
JobSnapshotTrigger Ein optionaler Trigger, der steuert, wann ein Snapshot des Jobs erstellt werden soll oder null.
JobTrigger Ein optionaler Trigger, der den Job wiederholt aufruft oder null (setzt intern BreakWithResult auf false).
Loggers Enthält eine Liste von externen Loggern für einen Job.
LogicalExpression Der logische Ausdruck, der durch eine JobList im LogicalTaskTree abgearbeitet wird.
LogicalName Der logische Name des Jobs.
NodeListUserControlPath Pfad zum dynamisch zu ladenden UserControl für eine NodeList.
Path Der Dateipfad zum gespeicherten Snapshot.
PhysicalPath Der Dateipfad zum gespeicherten Job.

StartCollapsed Bei True wird der Job beim Start zusammengeklappt angezeigt, wenn die UI dies unterstützt. Default: False
StartCollapsedString String-Property zur Vermeidung von Serialisierungsfehlern wegen Groß- Kleinschreibung
StructuredThreadLock Enthält eine Property "ThreadLocked" und einen optionalen "LockName" zur Differenzierung verschiedener Locking-Gruppen. Bei "ThreadLocked": True wird jeder Thread über die Klasse gesperrt, so dass nicht Thread-sichere Checker serialisiert werden; Ist darüber hinaus "LockName" gesetzt, wird nur gegen Checker mit dem gleichen LockName gesperrt. Default: False;
SubJobs Enthält eine Liste von SubJobs des aktuellen Jobs.
ThreadLockedString Wenn gesetzt, wird der Checker nicht gleichzeitig mit anderen Checkern ausgeführt, die auch ThreadLocked gesetzt haben. Wenn LockName gesetzt ist, wird nur gegen Checker mit dem gleichen LockName gesperrt. Default: False;
ThreadLockedXml Ist für die XML-Serialisierung notwendig.
TimeStamp Der Zeitpunkt, zu dem der Snapshot gespeichert wurde.
Trigger Ein optionaler Trigger, der den Job wiederholt aufruft oder null (setzt intern BreakWithResult auf false). Wenn JobTrigger nicht gesetzt ist, wird dieser verwendet. Bei Snapshots enthält dieser Trigger den Snapshot-Trigger.
TriggeredRunDelay Verzögert den Start eines Knotens (und InitNodes). Kann für Loops in Controlled-Jobs verwendet werden. Default: 0 (Millisekunden).
TriggeredRunDelayString String-Property zur Vermeidung von Serialisierungsfehlern wegen Int32 und String.
Triggers Enthält eine Liste von externen Triggern für einen Job.
UserControlPath Pfad zum dynamisch zu ladenden UserControl für eine JobList.
ValueModifiers Enthält eine Liste von Wertmodifikatoren für Checker.
Workers Enthält eine Liste von externen Workern für Checker eines Jobs.
