Changes in this version

  • The positioning of windows, WPF dialog checkers and messages on one and multiple screens has been completely revised.

  • The Vishnu-Help has bee integrated via F1 and menu item in the Context menu.

  • New Vishnu-parameter: HelpPreference

    • online: The online help is preferably loaded.
      This is the default setting.

    • local: The local help is preferably loaded.

  • The Plugin-directories below "VishnuHome/Vishnu" and "ReadyBin/Vishnu.bin" have been abolished. Affected user assemblies have been integrated into the other user assemblies.

    The UserAssemblyDirectory parameter should have become obsolete in the vast majority o cases. User assemblies are now found within the standard directory structures both in Vishnu-debugging, as well as from within Vishnu.bin.

  • The search for Jobs has been improved. It is now sufficient to specify the directories directly above the job; for example: DemoJobs\CheckServers, instead of %VishnuSourceRoot%\VishnuHome\Documentation\DemoJobs\CheckServers.

  • Bugfix: Workers were started multiple times with existing NodeConnectors on SingleNodes.

  • Bugfix: UserRun or UserBreak on a node with paused tree led to deadlock.
