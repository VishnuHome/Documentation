Version 5.3.0.3

Version 5.3.0.3 Heavy Duty (as of December 23, 2022)

Changes in this version

  • Implementation of a simple logging function for the results of any checker. CheckerHistoryLogger is itself referred to as Checker in a Job and in turn calls the actual checker, whose name was passed in the parameters, as a sub-checker.

  • Extensive revision of the job view:

    • Jobs are now displayed in the "Jobs" view in dynamic grids with a square layout. This has resulted in a better screen layout and improved handling when resizing.

    • Parameter StartWithJobs implemented.

    • Accelerating the preparation and positioning of the Vishnu-main window by parameter StartWithJobs and/or saved display settings.

  • Test jobs with 100 and 1000 checkers created and executed (...\TestJobs\HeavyDuty\).

  • All Vishnu nodes are now surrounded by a red or green border, depending on the logical state (true/false).

  • Context Menu now closes automatically when the mouse leaves the menu area.

  • Bugfix: WPFDialogCheckers always started in the first screen, even if Vishnu was running in the second screen.

  • Bugfix: Context Menu also opened when Ctrl or Shift was pressed together with the right mouse button.

  • All tests were run again and documented.
