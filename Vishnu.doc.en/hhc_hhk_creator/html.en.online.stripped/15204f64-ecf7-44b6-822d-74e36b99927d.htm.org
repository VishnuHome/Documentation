          <span class="keyword">SubWorkers</span> can only be given command line parameters (plain text)

          However, <strong>Vishnu</strong> extends the globally replacable parameters by some essential current
          runtime values:
        </p><ul><li><p><strong>"%Event%"</strong> = Name of the event that led to the worker being called
            </p></li><li><p><strong>"%Source%"</strong> = Source of the event (node in which the event first occurred)
            </p></li><li><p><strong>"%Sender%"</strong> = Node that currently calls the worker due to the event
            </p></li><li><p><strong>"%TreePath%"</strong> = complete path in the tree to the event source
            </p></li><li><p><strong>"%Timestamp%"</strong> = current date with current time in the format "dd.MM.yyyy HH.mm.ss"
            </p></li><li><p><strong>"%Logical%"</strong> = current logical value of the transmitter
            </p></li><li><p><strong>"%Exception%"</strong> = Exception.Message, if %Event% is "Exception", otherwise ""
            </p></li></ul><p>

          Own <span class="keyword">ParameterReaders</span> offer the possibility of passing additional, business-specific parameters to <strong>Vishnu</strong>.
          These parameters are then treated as normal <strong>Vishnu</strong>-parameters, which means,
          that they can also be subject of the <strong>Vishnu</strong>-parameter substitution.
        </p><p>
          Three components are required for this,
        </p><ol><li><p>
              Your own <strong>&lt;UserParameter-Name&gt;.dll</strong>.
              This <span class="keyword">DLL</span> must implement the
              <a href="T_Vishnu_Interchange_IParameterReader.htm">Vishnu.Interchange<span id="LST94805F17_1" data-languageSpecificText="cs=.|vb=.|cpp=::|nu=.|fs=."></span>IParameterReader</a>
              interface.
            </p><div class="alert"><table><tr><th><img src="../icons/AlertNote.png" alt="Hinweis" />Note</th></tr><tr><td>
            	<p><strong>Vishnu</strong> already provides a <span class="keyword">DemoParameterReader</span> and a
                Demo job <strong>CheckParameterReader</strong>.
                This will be discussed further below.
              </p></td></tr></table></div></li><li><p>
              Via the parameter <span class="keyword">UserAssemblyDirectory</span> a directory can be set
              in which <strong>Vishnu</strong> will search for your
              <a href="a3f9771a-ac24-46c0-97df-d2bde6a990e8.htm">Custom Checker DLLs</a>,
              <a href="74204d60-cbb3-4f38-8266-aa7af541143f.htm">Custom UserControls</a> or also
              <a href="f724b721-c805-4dbe-8db4-1529c0c68f3d.htm">Custom parameter reader</a>,
              in addition to the respective <span class="keyword">Job</span>-directories.
              The default is already set to <span class="keyword">UserAssemblies</span>.
            </p></li><li><p>
              Specification of the path to your own <span class="keyword">ParameterReader</span>.
              As already mentioned above <strong>Vishnu</strong> already provides a <span class="keyword">DemoParameterReader</span>
              and a <span class="keyword">Demo job</span> <strong>CheckParameterReader</strong>.
              The path can be found in the supplied <a href="9b9475bc-f8a9-4e7a-a84a-373276329b00.htm">Vishnu.exe.config.user</a> at
              <span class="keyword">"%UserAssemblyDirectory%\DemoParameterProvider.dll"</span>.
            </p></li></ol><p>
