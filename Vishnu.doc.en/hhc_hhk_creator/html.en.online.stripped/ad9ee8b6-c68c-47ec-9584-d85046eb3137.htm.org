            How Vishnu starts after the initial installation
          </div>
        </div>
        <p><strong>Vishnu</strong> is a modular system for
        <strong>Monitoring</strong> and <strong>Process
        Control</strong>.</p>
        <p>With <strong>Vishnu</strong> you can easily monitor and
        control any processes.</p>
        <p>These can be a wide variety of things, such as:</p>
        <ul>
          <li>
            <p>Are the machines running?</p>
          </li>
          <li>
            <p>Have the requirements for the next invoice run been
            met?</p>
          </li>
          <li>
            <p>Start the data backup!</p>
          </li>
          <li>
            <p>Inform mail recipients about errors and how to fix
            them!</p>
          </li>
        </ul>
        <p>But it can also be scientific evaluations, device
        monitoring in the internet of things, simply anything that
        meets simple basic requirements to be managed by
        <strong>Vishnu</strong>.</p>
        <div class="alert">
          <table>
            <tr>
              <th><img src="../icons/AlertNote.png" alt=
              "Hinweis">&nbsp;&nbsp;Note</th>
            </tr>
            <tr>
              <td>
                <p><span class="mediaInline"><img alt="no Drive D"
                src="../media/noDrive_D.png"></span> If
                <strong>Check_D</strong> looks something like this
                after installation,<br /> then your system does not
                currently have a D: drive.<br /> However, this is not
                important for further consideration.</p>
              </td>
            </tr>
          </table>
        </div>
        <p>As already outlined above, <strong>Vishnu</strong> is a
        software solution for monitoring and controlling any user
        process. Your processes can be processed simultaneously or
        interdependently (or both). You define the relationships
        using a simple description language (XML). The individual
        processes are grouped into jobs according to logical
        aspects. <strong>Vishnu</strong> displays the jobs and
        their actors clearly, either in a tree view or as lists.
        The actors within the jobs can trigger various other
        actions, e.g. messages, mails, other jobs, etc. Virtually
        everything that should be managed by
        <strong>Vishnu</strong> can be freely determined by you,
        including</p>
        <ul>
          <li>
            <p>Jobs - comprise several logically related actors,
            which in turn can also be jobs</p>
          </li>
          <li>
            <p>Checkers, Workers - any user processes (dll, exe)</p>
          </li>
          <li>
            <p>Triggers - restart checkers, workers or jobs, e.g.
            after a certain time</p>
          </li>
          <li>
            <p>Loggers, ValueModifiers, Connectors - further types of
            actors that will be described later. Details can be
            found at <a href=
            "bc0ffa08-c936-4fad-8fdb-dbd2279fc360.htm">Vishnu
            actors</a> and <a href=
            "0af46309-8c47-467f-a231-36615b7488d4.htm">Requirements
            for user modules</a>.</p>
          </li>
        </ul>
        <p><strong>Vishnu</strong> comes with a range of predefined
        demo jobs, checkers, triggers and workers so that you can
        get started straight away.<br>
        Try <strong>Vishnu</strong> directly with the <a href=
        "97ecd117-8937-4aca-bb5d-dfab02559f5e.htm">Quick Start</a>.</p>
        <p>Have fun and success.</p>
        <p>Erik Nagel</p>
