/></td><td class="titleColumn"><h1>Named Triggers and Loggers</h1></td></tr></table><div>
	<p><span class="keyword">Triggers</span> and <span class="keyword">Loggers</span> are usually anonymous,
        because they are directly assigned to <span class="keyword">Jobs</span> or <span class="keyword">Checkers</span>.
        This eliminates the need for them to be accessible via a <span class="keyword">LogicalName</span>.
        In some use cases, however, it may make sense to use <span class="keyword">Triggers</span> or <span class="keyword">Loggers</span>
        from different <span class="keyword">Checkers</span> or <span class="keyword">SubJobs</span> together.
        You can assign such <span class="keyword">Triggers</span> or <span class="keyword">Loggers</span>
        a <span class="keyword">LogicalName</span> and reference them via this <span class="keyword">LogicalName</span>
        in other <span class="keyword">Checkers</span> or <span class="keyword">SubJobs</span>.
      </p></div><div class="collapsibleAreaRegion" id="Beispiel"><span class="collapsibleRegionTitle" onclick="SectionExpandCollapse('IDDAB')" onkeypress="SectionExpandCollapse_CheckKey('IDDAB', event)" tabindex="0"><img id="IDDABToggle" class="collapseToggle" src="../icons/SectionExpanded.png" />Example</span></div><div id="IDDABSection" class="collapsibleSection"><p>
        Let's assume you would like to write to the shared log file from different <strong>Vishnu</strong>-nodes using the same parameters
        (node: generic term for all <a href="bc0ffa08-c936-4fad-8fdb-dbd2279fc360.htm">Vishnu actors</a>).
        You can achieve this by giving the <span class="keyword">Logger</span> a <span class="keyword">LogicalName</span>
        and referencing it from different <strong>Vishnu</strong>-nodes. The same applies to <span class="keyword">Triggers</span>
        that should start different nodes at the same time.
      </p><p>
          In the example below, the node with <span class="keyword">"Checker_1"</span> references
          <span class="keyword">"FirstNamedTimerTrigger"</span> and <span class="keyword">"FirstNamedLogger"</span>.
          More Checkers in the current Job or in SubJobs could do this as well. They would then listen to the same events
          and log with the same parameters as <span class="keyword">"Checker_1"</span>.
      </p><div class="mediaNear"><img alt="Named Trigger Logger" src="../media/NamedTriggerLogger.png" /></div><div class="alert"><table><tr><th><img src="../icons/AlertCaution.png" alt="Achtung" />Attention</th></tr><tr><td><p>
          
          The <span class="keyword">Logger</span> or <span class="keyword">Trigger</span> must be defined in a separate
          section (see example code) in the same <span class="keyword">Job</span> or in a higher level <span class="keyword">Job</span>,
          so that it can be found by all nodes using it.
